# Configuration files

The complete list of files sextant reads at runtime.

## `~/.config/sextant/sextantd.toml`

The daemon's configuration. The struct is `pkg/sextantd.Config` (`pkg/sextantd/config.go:17`). Full schema documented in [sextantd](../components/sextantd.md) §Configuration.

Created by `sextant init`. Required fields (asserted by `Config.Resolve()` at `pkg/sextantd/config.go:338-364`):

- `daemon.control_socket`
- `ca.key_path`, `ca.pub_path`
- `nats.data_dir`, `nats.operator_creds`
- `clickhouse.data_dir`
- `mcp.stdio_socket`
- `paths.templates_dir`, `paths.client_config`, `paths.runtime_file`

Defaults filled in by `Config.Resolve()` if missing: NATS/ClickHouse `listen_host` (`127.0.0.1`), ClickHouse `database`/`user` (`sextant`/`sextant`), MCP `http_host` (`127.0.0.1`), shipper `auto_supervise` (`true`), worktree `prune_interval` (`6h`), worktree `auto_prune` (`false`).

## `~/.config/sextant/client.toml`

Read by the Go client (`pkg/client/config.go`) and the TS client (`clients/typescript/src/config.ts`). Schema:

```toml
[nats]
url = "nats://127.0.0.1:4222"

[operator]
user        = "operator"
password    = ""                                    # OR
creds_path  = "~/.config/sextant/operator.creds"

[client]
connect_timeout = "10s"                             # Go: "10s"; TS: ms-int (10000)
request_timeout = "30s"
log_level       = "info"
```

> **`runtime.json` override**: the Go client (only) reads `~/.local/share/sextant/runtime.json` and uses its `nats_addr` instead of `nats.url` if present. This handles kernel-picked ports automatically.

## `~/.config/sextant/operator.creds`

NATS operator credentials. TOML:

```toml
user     = "operator"
password = "long-random-string"
```

Generated by `sextant init`, mode `0600`. The trust boundary for operator authority — anyone who can read this file has operator-level access to NATS (and therefore to the bus). See `specs/architecture.md` §10b.

## `~/.config/sextant/ca.key`, `ca.pub`

Ed25519 signing keypair, PEM-encoded. PKCS8 private, PKIX public. Generated by `sextant init`. Modes: `0600` for the private key, `0644` for the public. Loaded by `pkg/authjwt.LoadCA` at daemon start.

The private key signs every per-agent JWT. Lose it and every existing JWT becomes meaningless (re-spawn every agent).

## `~/.config/sextant/clickhouse.password`

The ClickHouse server password (separate file so it stays out of process listings). Read by `pkg/sextantd.ReadPasswordFile`.

## `~/.config/sextant/shipper.toml`

The shipper's configuration. Read both by the standalone `sextant-shipper` binary and (when `auto_supervise = true`) by `sextantd` to pass through to the supervised child. Schema documented in [shipper](../components/shipper.md) §Configuration.

## `~/.config/sextant/templates/*.toml`

Agent templates. One file per template; the file stem is the template name. Schema documented in [Templates](./templates.md).

`sextant init` creates `default.toml`. Operators add new files directly. `sextant templates reload` re-scans and pushes into NATS KV.

## `~/.config/sextant/secrets/`

Per-template-class subsets are surfaced to the container via `mounts = ["secrets"]`. Mounted read-only at `/run/sextant/secrets/`.

How an agent discovers which secrets it gets: the template's mount class determines the subset. The exact directory-to-template mapping is operator-controlled — there's no convention enforced by code at this snapshot beyond "directory contents are read-only inside the container."

## `~/.local/share/sextant/`

The data directory.

```
~/.local/share/sextant/
├── sextantd.sock              # operator control socket (mode 0600)
├── sextantd-mcp.sock          # MCP stdio socket (mode 0600)
├── runtime.json               # live ports + PID
├── nats/                      # JetStream data
├── clickhouse/                # ClickHouse data
├── shipper-buffer/buffer.db   # BoltDB spillover
└── worktrees/                 # default worktrees root (override via sextantd.toml)
```

## `~/.local/share/sextant/runtime.json`

Written by `sextantd` on startup; updated on subprocess restart. Contains live ports for NATS and ClickHouse plus the daemon PID and start time. Clients prefer it over `client.toml` (Go only at this snapshot).

## Environment variables

| Variable                  | Read by                | Effect                                                  |
|---------------------------|------------------------|---------------------------------------------------------|
| `SEXTANT_DOCKER_SOCKET`   | `pkg/containermgr:26`  | Override Docker socket auto-detection.                  |
| `SEXTANT_OPERATOR`        | TUIs                   | Override `os/user.Current().Username` for `ui_state` keys. |
| `SEXTANT_TEST_RUN_LABEL`  | `cmd/sextantd`         | Test harness marker stamped on spawned containers.      |

Container-side env vars are documented in [Sidecar image](../components/sidecar-image.md) §"Container environment variables".

## Files **not** in this build

- `~/.config/sextant/theme.toml` — referenced by `conventions/tui-conventions.md` for theme overrides. The structure is described; no shipped TUI reads it yet.
- `~/.config/sextant/cluster.toml` — referenced by `specs/architecture.md` §7 for multi-host federation. Multi-host is single-machine today, so this file isn't read.
